# SPECIFICATION.md — chiptune-studio（仮称）

8bit 音楽制作 Web アプリの仕様書。実装はこの文書のスコープを厳守する。

**作業名（仮）: `chiptune-studio`。** 正式名称が決まったら repo 名・package name・UI のワードマークをまとめて置換する。

---

## 1. 概要

ブラウザで動く 8bit（チップチューン）音楽制作アプリ。ファミコン・スーパーファミコン・ゲームボーイの音を Web Audio API で再現し、8 小節のループをステップシーケンサーで打ち込み、WAV としてエクスポートできる。PWA としてインストール・オフライン動作可能。

- **ターゲット**: チップチューンを気軽に作りたい人。DAW の知識は前提にしない
- **プラットフォーム**: デスクトップブラウザ優先（Chrome / Safari / Firefox 最新版）。主要操作はタッチでも可能にするが、モバイル最適化は非目標
- **サーバ**: なし。完全クライアントサイド。データは localStorage とファイル入出力のみ

---

## 2. 技術スタック

- React 18 + TypeScript（strict）+ Vite
- Tailwind CSS（ドット調テーマは CSS 変数で定義し Tailwind theme に流し込む）
- 状態管理: Zustand（プロジェクトデータを 1 ストアで保持）
- サウンド: Web Audio API 素の API（Tone.js 等のラッパーは使わない。チップ音源の再現に低レベル制御が必要なため）
- PWA: vite-plugin-pwa
- テスト: Vitest（音の聴感は自動テスト不能なので、対象はスケジューラ計算・データモデル・入出力）

## 3. コア概念とデータモデル

### 3.1 プロジェクト

single source of truth はこの `Project` オブジェクト。UI・再生・エクスポートはすべて同じ 1 つのデータを見る。

```ts
type ConsoleMode = 'famicom' | 'superfamicom' | 'gameboy';

interface Project {
  version: 1;              // スキーマバージョン（マイグレーション用）
  title: string;
  bpm: number;             // 40–240
  consoleMode: ConsoleMode;
  tracks: Track[];         // 固定 4 トラック（piano / guitar / bass / drums）
}

interface Track {
  id: 'piano' | 'guitar' | 'bass' | 'drums';
  volume: number;          // 0–1
  mute: boolean;
  solo: boolean;
  notes: Note[];
}

interface Note {
  step: number;            // 0–127（開始位置）
  pitch: number;           // MIDI ノート番号。drums はレーン番号（§5.2）
  length: number;          // ステップ数（drums は常に 1）
  velocity: number;        // 0–1
}
```

### 3.2 タイムライン

- **8 小節・4/4 拍子固定**。1 小節 = 16 ステップ（16 分音符グリッド）→ 全体 128 ステップ
- 再生は常にループ（128 ステップで先頭に戻る）
- 小節数・拍子の変更は非目標

### 3.3 コンソールモード

音色の「ハード」を切り替えるグローバル設定。トラックごとではなくプロジェクト全体で 1 つ。

| モード | 再現方針 |
|---|---|
| famicom | 矩形波（デューティ比 12.5/25/50%）・三角波・ノイズ。エンベロープは段階的な音量変化 |
| gameboy | 矩形波 2 種 + 4bit 波形メモリ風 + ノイズ。全体に 4bit 量子化の質感（ビットクラッシュ） |
| superfamicom | 上記より柔らかい音。波形にローパスをかけ、擬似的に 32kHz サンプル感を出す |

厳密なハードウェアエミュレーションは目標にしない。「それっぽく聴こえる」ことがゴール。

## 4. サウンドエンジン（`src/audio/`、UI 非依存）

### 4.1 音源

Web Audio API の `OscillatorNode`（カスタム PeriodicWave 含む）+ `AudioBufferSourceNode`（ノイズ）+ `GainNode`（エンベロープ）で構成。

楽器とチャンネルの対応（famicom モードの例）:

| 楽器 | 波形 | エンベロープ |
|---|---|---|
| piano | 矩形波 50% | 速いディケイ（ピアノらしい減衰） |
| guitar | 矩形波 25% | 中程度のディケイ + 軽いビブラート |
| bass | 三角波（1 オクターブ下で発音） | サステイン型 |
| drums | ノイズ + ピッチ下降する三角波（キック） | ワンショット |

### 4.2 ドラムキット

drums トラックは 4 レーン固定:

| レーン (pitch) | 音 | 作り方 |
|---|---|---|
| 0 | キック | 三角波の急速ピッチ下降 + 低域ノイズ短発 |
| 1 | スネア | 中域ノイズ + 短い矩形波 |
| 2 | ハイハット（閉） | 高域ノイズ極短 |
| 3 | ハイハット（開） | 高域ノイズ長め |

### 4.3 スケジューラ

- lookahead 方式（setInterval 25ms で先読み 100ms 分の `AudioContext.currentTime` 基準スケジューリング）。`requestAnimationFrame` は UI の再生ヘッド表示のみに使う
- BPM 変更は次のステップから反映
- メトロノーム: 拍頭にクリック音（1 拍目は高い音、2–4 拍目は低い音）。オン/オフ切替可。エクスポートには含めない

### 4.4 エクスポート用レンダリング

`OfflineAudioContext` で同じエンジンコードを走らせて 8 小節分をレンダリングし、WAV に変換する。**リアルタイム再生とオフラインレンダリングでコードパスを分岐させない**（AudioContext を引数で受け取る設計にする）。

## 5. UI 仕様

### 5.1 画面構成（1 画面完結、ルーティングなし）

```
┌──────────────────────────────────────────┐
│ ヘッダー: ワードマーク / プロジェクト名 /       │
│           コンソールモード切替 / 保存・読込・書出 │
├──────────────────────────────────────────┤
│ トランスポート: ▶ ■ / BPM / メトロノーム /      │
│                再生位置表示（小節:拍）           │
├──────────────────────────────────────────┤
│ トラックタブ: [ピアノ][ギター][ベース][ドラム]    │
│  各タブに音量スライダー / M / S                 │
├──────────────────────────────────────────┤
│ シーケンサー本体（選択中トラックを表示）           │
│  - ピッチ系: ピアノロール                       │
│  - ドラム: 4 レーンのステップグリッド             │
└──────────────────────────────────────────┘
```

### 5.2 シーケンサー操作

**ピアノロール（piano / guitar / bass）**

- 縦軸: 音高。表示範囲は C2–C6（bass は C1–C4）。縦スクロール可
- 横軸: 128 ステップ。小節境界と拍を罫線の濃淡で区別
- クリックでノート追加（デフォルト長 1 ステップ）、ノート右端ドラッグで長さ変更、ノートクリックで削除、ドラッグで移動
- ノート追加時にその音をプレビュー発音する

**ドラムグリッド**

- 4 レーン × 128 ステップの ON/OFF グリッド。クリックでトグル
- ドラッグで連続入力可能

**共通**

- 再生中はプレイヘッド（縦線）が移動し、自動横スクロールで追従
- スペースキーで再生/停止

### 5.3 ドット調ビジュアル

- フォント: 見出し・UI ラベルは DotGothic16（Google Fonts、PWA のためセルフホスト）。数値は等幅ピクセルフォント
- 角丸なし。ボーダーは 2px 単調。影は 4px オフセットのベタ塗り（ぼかしなし）
- ボタン押下は 2px 沈み込みで表現
- カラーパレットはゲームボーイ風 4 階調グリーンを基本とし、famicom / superfamicom モードでアクセント色が変わる（背景・文字は共通）
- `image-rendering: pixelated`、アイコンは自作の SVG ドット絵（外部アイコンライブラリは使わない）
- グラデーション・ブラー・アンチエイリアスな装飾は禁止

## 6. 保存・読み込み・エクスポート

| 機能 | 形式 | 詳細 |
|---|---|---|
| 自動保存 | localStorage | 変更のたびに debounce 1 秒で保存。起動時に復元 |
| プロジェクト保存 | `.json` ダウンロード | `Project` をそのまま書き出し |
| プロジェクト読み込み | `.json` ファイル選択 | version チェック + バリデーション。不正なら理由を表示して拒否 |
| 音声エクスポート | `.wav`（44.1kHz / 16bit / ステレオ） | OfflineAudioContext でレンダリング。ループ 1 周分（8 小節）+ リリース余韻 1 秒 |

MIDI エクスポート・MP3 エクスポートは非目標（将来検討）。

---

## 7. PWA 要件

- vite-plugin-pwa で manifest + service worker を生成
- 全アセット（フォント含む）をプリキャッシュし、**完全オフラインで全機能が動く**こと
- manifest: `display: standalone`、テーマカラーはドット調パレットに合わせる。アイコンはドット絵で 192/512px
- 更新戦略: 新バージョン検出時にトースト表示 →ユーザー操作でリロード（自動リロードで編集中データを飛ばさない）

## 9. 実装順（feature slices）

1 セッション = 1 スライス = 1 ブランチ。各スライスが緑（typecheck / lint / build 通過 + 動作確認）になったらコミット。

1. **雛形**: Vite + React + TS + Tailwind + ドット調テーマ基盤 + 共有 UI 部品（ボタン・スライダー）
2. **model 層**: `Project` 型・初期プロジェクト生成・バリデーション + Zustand ストア
3. **サウンドエンジン最小版**: famicom モードの 4 楽器を単音で鳴らせる（試聴ボタンで確認）
4. **スケジューラ + トランスポート**: 再生/停止・BPM・ループ・プレイヘッド・メトロノーム
5. **ドラムグリッド**: 打ち込み → 再生が通る（最初の「曲が作れる」瞬間）
6. **ピアノロール**: piano / guitar / bass の打ち込み。追加・削除・長さ変更・移動
7. **トラック操作**: タブ切替・音量・ミュート・ソロ
8. **保存/読込**: localStorage 自動保存 + JSON 入出力
9. **WAV エクスポート**: OfflineAudioContext レンダリング
10. **コンソールモード**: gameboy / superfamicom の音色バリエーション
11. **PWA 化**: manifest・service worker・オフライン確認・更新トースト
12. **仕上げ**: キーボードショートカット・音量バランス調整・ドット絵アイコン

MVP はスライス 9 まで。10 以降は MVP 後の拡張として扱う（ただし consoleMode はデータモデルに最初から含める）。

## 10. 非目標（やらないこと）

- 8 小節を超える楽曲・ソングモード・複数パターン
- 拍子・音階グリッドの変更
- エフェクト（リバーブ・ディレイ等）
- MIDI 入出力・外部キーボード対応
- 録音・サンプリング・音声ファイル読み込み
- ユーザーアカウント・クラウド保存・共有機能
- モバイル UI の本格最適化（タッチで一応操作できれば良い）
- ハードウェアの厳密なエミュレーション

---

## 11. 完了の定義

- `npm run typecheck`・`npm run lint`・`npm run build` が通る
- 実ブラウザで音を鳴らして動作確認済み（音に関わる変更は必ず耳で確認する）
- エクスポートした WAV が外部プレイヤーで再生できる
- PWA スライス以降: オフライン（DevTools の offline モード）で全機能が動く

---

## 12. 既知のリスクと方針

- **AudioContext の自動再生制限**: 初回のユーザー操作（再生ボタン等）で `resume()` する。ページロード時に音を出さない
- **タイミング精度**: UI スレッドのブロックに備え lookahead を十分取る。将来必要なら AudioWorklet 化を検討（MVP では見送り）
- **Safari 対応**: OfflineAudioContext・PeriodicWave の挙動差は実機確認する。動かない装飾より動く音を優先
